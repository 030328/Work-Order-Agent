# 作品集展示指南

这份文档用于把项目讲成一个适合 Java/后端/全栈实习投递的作品，而不是只展示“用了很多技术”。

## 一句话介绍

这是一个基于 Spring Cloud 的企业级智能工单系统，覆盖工单提交、AI 分析、人工处理、状态流转、评论协作、SLA 超时升级和知识库沉淀。

## 推荐演示路径

1. 使用 `wangwu/admin123` 登录，创建一条 BUG 工单。
2. 展示 AI 自动生成摘要、情绪、建议分类和解决方案。
3. 如果 AI 无法解决，提交人点击“转人工处理”。
4. 使用 `zhangsan/admin123` 登录，认领工单，添加内部备注，提交处理意见。
5. 切回提交人，查看评论和流转记录，选择确认完成或驳回。
6. 展示关闭后知识库沉淀，以及下次 AI 分析如何参考历史经验。

## 面试时重点讲什么

- **权限不是只做登录**：网关会清理伪造用户头，业务服务从 JWT 解析身份，工单对象级权限控制谁能看、谁能改。
- **工作流不是硬编码 if else**：工单服务更新状态前会调用 Workflow 服务校验 `wf_transition` 规则。
- **评论和流转记录职责不同**：评论用于协作沟通，流转记录用于审计追踪。
- **AI 不是孤立聊天框**：AI Agent 参与工单分析和知识库检索，并在工单关闭后沉淀经验。
- **SLA 有业务闭环**：创建时计算 deadline，定时扫描超时工单，自动升级到人工队列。

## 可以继续优化的方向

- 接入 MinIO，把附件元数据升级为真实文件上传和下载。
- 为前端增加 E2E 测试，例如 Playwright 登录、创建、评论、关闭工单流程。
- 增加 outbox pattern，保证数据库状态变更和 MQ 事件一致性。
- 增加工作流规则管理页面，让管理员动态调整状态流转。
- 增加 OpenAPI/Swagger 文档，方便接口调试和面试展示。

## 简历写法参考

- 基于 Spring Cloud Alibaba 搭建智能工单系统，拆分 Gateway、User、WorkOrder、Workflow、AI Agent 等服务。
- 设计 JWT 鉴权和 RBAC 权限模型，实现工单对象级访问控制和服务间内部 Token。
- 设计工单状态机流程，支持创建、AI 处理、转人工、认领、解决、驳回、关闭和流转审计。
- 接入 DashScope 与 Milvus，实现基于 RAG 的历史工单检索与智能处理建议。
- 使用 Docker Compose 编排 MySQL、Redis、Nacos、RocketMQ、Elasticsearch、Milvus 等基础设施。
