# 前端展示优化说明

本文档说明本次前端页面优化的目标、改动范围和面试表达方式。

## 1. 优化目标

原页面已经能完成登录、工单列表、创建工单、详情查看、AI 对话和知识库查询，但展示层更偏 demo。对于“企业级 Agent 工单系统”这个简历项目，前端需要体现三个能力：

- 信息层级清楚：用户一眼能看到待处理工单、SLA 风险、状态和负责人。
- 流程感更强：创建、流转、AI 分析、知识库召回之间有关联。
- 工程质感更好：页面布局统一，移动端不明显错位，按钮和空状态完整。

## 2. 页面改动

| 页面 | 优化点 |
| --- | --- |
| 登录页 | 改成左右分栏，突出系统定位和默认测试账号，登录卡片更聚焦。 |
| 主布局 | 新增企业后台式侧边栏和顶部标题区，不同路由展示不同页面说明。 |
| 工单列表 | 增加指标概览、筛选工具栏、状态/优先级标签和空状态。 |
| 创建工单 | 改成表单 + 流程侧栏 + 创建结果，能展示 AI 分类和 SLA 截止时间。 |
| 工单详情 | 增加摘要区、基础信息、操作区、AI 分析区，减少信息堆叠。 |
| AI 对话 | 改成消息流 + 快捷问题 + 能力侧栏，展示 Agent 辅助排障场景。 |
| 知识库 | 增加检索表单、结果表格、详情弹窗和数据质量侧栏。 |

## 3. 为什么这样优化

企业级后台页面的重点不是花哨，而是可扫描、可操作、可解释。本次优化尽量保持克制：

- 使用浅色背景、白色工作区和小圆角，接近真实 ToB 系统。
- 用标签、表格、描述列表承载结构化数据，便于面试官快速理解业务字段。
- 把 AI、工作流、SLA 的结果放进业务页面，而不是单独做展示页，这样更像一个真正落地的系统。
- 前端 API 地址改为 `VITE_API_BASE_URL`，配合后端配置外置化，方便本地、Docker、服务器部署切换。

## 4. 验证方式

前端生产构建：

```powershell
cd wo-frontend
npm run build
```

本地预览开发页面：

```powershell
cd wo-frontend
node .\node_modules\vite\bin\vite.js --host 127.0.0.1 --port 5173
```

浏览器打开：

```text
http://127.0.0.1:5173/
```

登录测试账号：

```text
admin / admin123
```
